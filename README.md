## 🔄 Flujo de Trabajo Usado
Implementamos un flujo de trabajo basado en ramas independientes para mantener la estabilidad del código:

1. **Rama `main`**: Contiene el código completamente estable y aprobado.
2. **Ramas de Desarrollo**:
   * `dev`: Dedicada a la construcción de la estructura lógica y maquetación HTML.
   * `diseno`: Enfocada exclusivamente en los estilos CSS, diseño visual y experiencia de usuario.
3. **Pull Requests (PR)**: Los cambios de la rama `dev` se integraron a `main` mediante un Pull Request que requirió la revisión y aprobación de otro integrante del equipo.
4. **Resolución de Conflictos**: Se generó un conflicto real en el archivo `index.html` al modificar la misma línea de código en `main` y `diseno`. El conflicto fue resuelto manualmente unificando los criterios del equipo antes de realizar la integración final.
